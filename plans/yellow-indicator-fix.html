Yellow Indicator Fix

TerminalPane.tsx — agent running-state detection

The Desired Behaviour

Yellow blinking → The agent is doing anything: thinking, calling tools, generating output, rendering its response
Green solid → The agent is idle at its prompt, waiting for the user to type

The only reliable signal we have is PTY output: when the agent writes anything to the terminal, it is active. When the terminal has been silent for a few seconds, the agent is idle at its prompt. That's the entire model. Everything else is implementation detail.

The Bug

Root cause The gate in onPtyData blocks re-arming after the idle timer fires

When PTY data arrives, the handler checks a flag before deciding whether to light up the indicator:

const removeDataListener = window.api.onPtyData(session.id, (data) => {
  touchSessionActivity()
  if (agentTaskInFlightRef.current) {    // ← the broken gate
    markAgentRunning(agentProcessRef.current)
  }
  ...

agentTaskInFlightRef.current starts true when a task begins, but the idle timer sets it to false after 6 seconds of silence. Once it goes false, PTY output from Claude never turns the dot yellow again — not for the rest of that task, and not for any later tasks in the same session.

The idle timer was intended to detect "Claude returned to its prompt." Instead it becomes a one-way latch that permanently disables the indicator mid-task.

What this looks like in practice

User submits task yellow ✓
Enter → markAgentRunning() → agentTaskInFlightRef = true, idle timer starts at 6 s.
Claude generates initial output yellow ✓
PTY data → agentTaskInFlightRef.current is true → timer resets each time.
Claude calls a tool — PTY goes quiet yellow ✓
No output while waiting for tool or API. Timer counting down. Still correct for now.
6-second timer fires (tool still running) green ✗
markAgentIdle() → agentTaskInFlightRef = false. Dot turns green. Claude is still mid-task.
Tool finishes — Claude resumes, PTY output flows stays green ✗
agentTaskInFlightRef.current is false → gate blocks → markAgentRunning() never called. Dot stays green for the rest of the task, through all further tool calls.

The Fix

Mental model after fix

PTY data arrives → if an agent session is active → dot goes yellow + timer resets.
Timer fires after 8 seconds of silence → dot goes green.
More PTY output → dot immediately snaps back to yellow.

The timer is now just a debounce, not a one-way latch. Its only job is: "if nothing has been printed for 8 seconds, assume Claude is at its idle prompt."

Change 1 — Fix the gate the real fix

TerminalPane.tsx · onPtyData handler, ~line 823
const removeDataListener = window.api.onPtyData(session.id, (data) => {
  touchSessionActivity()
  if (agentTaskInFlightRef.current) {
  if (agentSessionRef.current) {
    markAgentRunning(agentProcessRef.current)
  }
  ...

Instead of gating on the now-stale agentTaskInFlightRef, check agentSessionRef.current — which is true for the entire lifetime of an agent process in this terminal. Any PTY output → re-arm. Timer fires after silence → idle. PTY output again → re-arm. The latch is gone.

Change 2 — Adjust idle delay to 8 s debounce tuning

TerminalPane.tsx · line 13
const AGENT_IDLE_DELAY_MS = 6_000
const AGENT_IDLE_DELAY_MS = 8_000

With the gate fixed, the timer value only controls how long after Claude's last output the dot turns green. 8 seconds gives a comfortable buffer for brief pauses in output (between paragraphs, streaming chunks, etc.) without making the dot feel laggy when Claude truly finishes. If a tool call is silent for longer than 8 s, the dot briefly turns green — then immediately snaps back yellow when Claude resumes. That's correct behaviour.

agentTaskInFlightRef is now write-only — its value is set in three places but never read for any decision after Change 1. Remove it entirely to avoid confusion.

Summary of Changes

  • → Gate fix — onPtyData: agentTaskInFlightRef.current → agentSessionRef.current. This is the fix. One word changed.
  • → Idle delay — AGENT_IDLE_DELAY_MS: 6 000 → 8 000 ms. Now a pure debounce, not a one-way latch.
  • ✗ Remove agentTaskInFlightRef — ref declaration + 3 write sites. Dead code after the gate fix.
  • — No changes to Sidebar.tsx, App.tsx, or CSS. The indicator logic and animation are correct; the bug was entirely in how agentState got set.

How to Verify

  1. Open a tab, run claude or codex, give it a multi-step task (e.g. "read all TypeScript files and summarise the architecture")
  2. Watch the sidebar dot — it should blink yellow continuously throughout, including during silent tool-call gaps
  3. If a gap causes a brief flicker to green, it should snap back yellow the moment Claude prints its next line
  4. After Claude returns to its prompt, dot turns green within ~8 seconds of silence
  5. Start a second task without relaunching — dot should go yellow again immediately as Claude responds
  6. Confirm a plain shell tab (no Claude) is unaffected — dot should stay green/red as usual
